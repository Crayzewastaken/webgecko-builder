WebGecko Process Audit & Action Plan

Core Pipeline Area Risk Level Observed Key Vulnerability Immediate Fix Required Impact & Rationale
1 Client Portal Auth Fixed & Retested Unauthenticated client content routes (/api/client/content GET/POST) Implement wg_client_slug cookie authentication matching the pattern in client-login.ts Allows any internet user to read queue or inject spam/malicious briefs if they guess or scrape the slug.
2 Intake Pipeline Fixed & Retested jobId generated via Date.now(), risk of concurrent collisions Replace Date.now() with crypto.randomUUID() for absolute database uniqueness Simultaneous double-submissions within the same millisecond overwrite the first client's DB record.
3 Admin Panel Auth Fixed & Retested Admin login comparison timing leaksexpected password length Implement hash-then-compare (bcrypt/PBKDF2) instead of raw length checks Timing oracle allows attackers to isolate the password length, narrowing down brute-force permutations.
4 Core Orchestration Fixed & Retested Inngest monolithic route.ts (2,180 lines) prone to NTFS file truncation Step 5 (~630 lines) extracted into lib/pipeline-step5.ts — route.ts reduced from 2,466 → 1,854 lines. applyStep5CodeFixes() delegates all CTA wiring, maps, SEO meta & social injection. Windows/NTFS write processes on large files corrupt them mid-statement, blocking Vercel builds constantly. Risk now eliminated by splitting the heaviest block.
5 Billing & Revenue Fixed & Retested Recurring host subscriptions not tied to service deployment Integrate a Square webhook receiver that automatically suspends Vercel alias on default Clients who cancel cards or recurring subscriptions continue to receive full live hosting indefinitely.
6 Analytics Engine Fixed & Retested getTopPages() fetches all database rows to count in Node.js memory Refactor query to use Supabase SQL GROUP BY / COUNT query directly on database At scale (e.g. 10,000 page views), reading every raw row into RAM crashes the serverless API instance.
7 Pricing Logic Fixed & Retested calculatePrice() logic is duplicated across 3 separate files Consolidate pricing calculations into a unified lib/pricing.ts helper module Any future price tweaks must be updated in 3 separate places. Discrepancies cause billing inaccuracies.
8 Client Security Fixed & Retested Plaintext Square & Stripe OAuth keys stored in jobs table Encrypt all payment tokens at rest using the existing PAYLOAD_ENCRYPTION_KEY A leak of the database exposes live transaction credentials for every single customer site.
9 SEO Optimization Fixed & Retested Google Indexing API pings .vercel.app instead of custom domains Modify requestGoogleIndexing to ping the canonical domain once it is provisioned Causes Google to index temporary Vercel subdomains, degrading custom domain ranking authority.
10 Log Separation Fixed & Retested Unbounded pipeline logs were appended directly inside Supabase jobs metadata JSON blob appendPipelineLog() now INSERTs to dedicated pipeline_logs table (O(1) write). Migration 20260523_pipeline_logs_table.sql deployed. Legacy RMW fallback retained for existing rows. Appends caused massive DB block bloat, slowing down dashboard load and job reads as logging grew. Now O(1) per write with a 90-day TTL index.

Process Pipeline: Form Submission to Live Release

The Process Flow

Step 2: Payment

Step 3: Compilation

⚙️ Automation Pipeline

10-step serverless build: creates booking schedule, synthesizes layout, runs copywriting, injects features, and verifies structure.

Step 4: Smoke Test

🔍 Code Audit & Validate

Self-healing validation checks 15+ metrics (FAQ accordion, footers, booking frame, forms). Re-audits & fixes page if needed.

Step 5: Cloud Deploy

🚀 Vercel Deploy & Index

Launches custom isolated Vercel project, registers the stable server alias, and triggers a direct search engine index ping.

Step 6: Handover

🔑 Client Portal Release

Auto-release cron unlocks the portal 10 days later, emailing secure preview links. Client approves & triggers custom domain mapping.
